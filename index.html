About Joined Node

It's a familiar story. You get a great idea for an app, and start building it... But wait...

Do you really need the hassle of setting up an entire server when all you want to do is have an HTML page that sends an SMS message? or an email? or posts to your Slack channel?

There needs to be a simpler way to do this...

This is where Joined Node comes in handy...


At the heart of Joined Node, are Recipes, a Recipe is a snippet of code that can be called by using a simple HTTP request, either directly inside a browser, or from inside your code, regardless of language or platform.

These Recipes can be very simple, or they can be extended as much as you wish. You can use them to handle incoming SMS messages; send outgoing SMS messages or emails; send a notification to Slack; process Stripe payments; save to a database somewhere. Joined Node is all about quick recipes that do one purpose and only one purpose.


These Joined Node Recipes are also known as microservices, each recipe does one job, and does it well. This also helps keep your code clean and organized, while also helping you build out your apps faster when you don't have to worry about huge backend systems.
